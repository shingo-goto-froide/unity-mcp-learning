# テストフェーズ：バグ調査・修正

以下のファイルをすべて読み込んでください：
- `Assets/Docs/PROJECT.md`（既知バグ・TODO・プロジェクト固有の注意事項を確認）
- `Assets/Docs/仕様書_*.md`（最新版）
- `Assets/Docs/設計書_*.md`（最新版）

**バグ対応フロー（CLAUDE.md 準拠）：**

| バグの種類 | 対応方法 |
|---|---|
| 軽微なバグ（仕様・設計に影響しない） | CLI即修正 → PROJECT.md の修正記録に残す |
| 仕様・設計に影響するバグ | 仕様書を更新 → 設計フェーズへ戻る |

**調査手順：**
1. エラー内容・再現条件を整理する
2. 関連するスクリプトを読み込む
3. 仕様書・設計書と照合して原因を特定する
4. 軽微バグなら即修正、仕様変更が必要なら設計フェーズへ戻ることを提案する

> ⚠️ プロジェクト固有の既知バグ・禁止事項・注意事項は PROJECT.md を参照すること。

修正後は PROJECT.md の変更履歴に追記すること。
