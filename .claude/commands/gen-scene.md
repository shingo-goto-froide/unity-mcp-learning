# 実装フェーズ：シーン構築

## スコープ指定

このコマンドは引数でスコープを指定できます：

| 引数 | 説明 | 使用タイミング |
|---|---|---|
| `core` | コアループが確認できる最小限のシーン構築 | コアループ実装フェーズ（プレイテスト前） |
| `full` | 残りのシーン・UIを構築（core で構築済みのものはスキップ） | 残り実装フェーズ（プレイテスト後） |
| （引数なし） | 全シーンを一括構築 | スコープ分割が不要な小規模プロジェクト |

### スコープの判断基準

**core に含めるもの：**
- GameScene（コアループが動作する最小構成）
- コアループに必要な最低限のUI（操作ボタン・基本表示）
- EventSystem・Camera・Manager系

**full に含めるもの：**
- TitleScene・ResultScene等
- スコアボード・設定画面等の追加UI
- 演出用オブジェクト
- その他コアループ動作に不要なもの

> ⚠️ 判断に迷う場合はシーン構成書を見て、人に確認してから決める。

---

以下のファイルをすべて読み込んでください：
- `Assets/Docs/PROJECT.md`（プロジェクト固有の制約・注意事項を把握する）
- `Assets/Docs/シーン構成書.md`
- `Assets/Docs/設計書_*.md`（最新版）

読み込んだ後、シーン構成書に従いシーンを構築してください。
**スコープが `core` または `full` の場合、該当するシーン・オブジェクトのみを対象にすること。**

**構築手順：**
1. シーン構成書のシーン一覧に記載されたシーンを対象にする
2. Hierarchy に従い GameObject を作成
3. コンポーネント一覧に従いコンポーネントをアタッチ
4. Inspector 参照設定に従いフィールドを接続
5. Canvas / CanvasScaler 設定を行う

> ⚠️ シーン名・フォント・禁止コンポーネント・レイアウト詳細等のプロジェクト固有の制約は PROJECT.md の注意事項を参照すること。

---

構築完了後：
- `core` の場合：「コアループのシーン構築が完了しました。Unityで実行して動作確認してください」と通知する。
- `full` の場合：「残りのシーン構築が完了しました。人の確認後に Git commit してください」と通知する。
- 引数なしの場合：「シーン構築が完了しました。人の確認後に Git commit してください」と通知する。
