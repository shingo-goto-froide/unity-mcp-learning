# 実装フェーズ：スクリプト一括生成

## スコープ指定

このコマンドは引数でスコープを指定できます：

| 引数 | 説明 | 使用タイミング |
|---|---|---|
| `core` | コアループに必要なスクリプトのみ生成 | コアループ実装フェーズ（プレイテスト前） |
| `full` | 残りの全スクリプトを生成（core で生成済みのものはスキップ） | 残り実装フェーズ（プレイテスト後） |
| （引数なし） | 全スクリプトを一括生成 | スコープ分割が不要な小規模プロジェクト |

### スコープの判断基準

**core に含めるもの：**
- ゲームの基本ルールを動かすために必須のクラス（GameManager等）
- コアループを構成するフェーズ・ターン管理
- 最低限の入力・操作系
- 上記が依存する Enum・ScriptableObject・データクラス

**full に含めるもの：**
- UI関連（スコアボード・設定画面等）
- AI・CPU対戦ロジック
- 演出・エフェクト・サウンド関連
- タイトル画面・リザルト画面
- その他コアループ動作に不要なもの

> ⚠️ 判断に迷う場合は設計書の依存関係ツリーを見て、人に確認してから決める。

以下のファイルをすべて読み込んでください：
- `Assets/Docs/PROJECT.md`（プロジェクト固有の制約・注意事項を把握する）
- `Assets/Docs/仕様書_*.md`（最新版）

## 依存関係の分析と並列実行計画

> ⚠️ 以下の並列グループは設計書完成後にこのプロジェクト用に更新すること。
> 更新方法：「設計書を読んで並列グループを提案して」とClaude Codeに依頼する。

設計書の依存関係ツリーを確認し、以下の順序で生成してください。
**同じステップ内のグループは互いに独立しているため、並列（同時）で生成すること。**
**スコープが `core` または `full` の場合、該当するクラスのみを対象にすること。**

```
ステップ1（並列）：Enum・ScriptableObject・静的クラス
  └── グループA：（設計書完成後に記入）

ステップ2（並列）：純粋C#クラス（MonoBehaviour非依存）
  └── グループB：（設計書完成後に記入）
  └── グループC：（設計書完成後に記入）

ステップ3（並列）：Core MonoBehaviour
  └── グループD：（設計書完成後に記入）

ステップ4（並列）：UI・AI・その他MonoBehaviour
  └── グループE：（設計書完成後に記入）
  └── グループF：（設計書完成後に記入）
```

---

## 実装の注意事項

- コンパイルエラーはその場で自己修正すること
- MonoBehaviour の Singleton は `static Instance` パターンで実装
- `core` スコープの場合、ステップ4のUI・AI系は基本的にスキップ
- `full` スコープの場合、既に生成済みのファイルが存在すればスキップ（上書きしない）

> ⚠️ using指定・使用禁止コンポーネント・アセットパス等のプロジェクト固有の制約は PROJECT.md の注意事項を参照すること。

- `full` の場合：「残りのスクリプト生成が完了しました。人の確認後に Git commit し、次は /gen-scene full でシーン構築してください」と通知する。
- 引数なしの場合：「スクリプト生成が完了しました。人の確認後に Git commit し、次は /gen-scene でシーン構築してください」と通知する。
