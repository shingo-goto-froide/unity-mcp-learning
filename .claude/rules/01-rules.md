# Rules（Claudeへの行動ルール）

> Claude Code CLI・Claude Desktop 共通のルール。
> セッション開始時に必ず読み込むこと。

## ✅ 必ず守ること

- **ドキュメントを読んでからコードを書く**（`Assets/Docs/` が存在する場合は必ず先に読む）
- **仕様変更を伴う実装は人の承認を得てから行う**
- **コミットは人の指示があるまで自動実行しない**
- **スクリプト生成前に設計書の存在を確認する**
- **実装とドキュメントが乖離していたら必ず指摘する**
- **エラーが出たら原因を説明してから修正する**（黙って直さない）
- **不明点は作業前に確認する**（推測で進めない）

## ❌ やってはいけないこと

- `SampleScene` を使用・編集する
- `StandaloneInputModule` を使う（`InputSystemUIInputModule` を使うこと）
- 人の確認なしに既存スクリプトを削除・大幅リファクタリングする
- プレイテスト前に演出・アニメーション実装を進める
- コアループが未確認のまま残り実装に進む
- git push を自動実行する

## 🔄 判断に迷ったら

- 仕様書に記載がない → 人に確認してから進める
- バグ修正でインターフェースが変わる → 設計書更新が必要か確認する
- 実装量が多い → 並列実行できるタスクを分けて提案する
