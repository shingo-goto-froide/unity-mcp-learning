# リソース争奪ボードゲーム 仕様書 v2.0
> 実装コードと同期済み（2026-04-07 最終更新）

---

## ゲーム概要

2人対戦のボードゲーム風ストラテジーゲーム。
リソースの獲得・配置を通じた戦略的な攻防が楽しめる。
AI対戦・オンライン対戦（予定）に対応。

### 想定対戦時間
- **目標：5〜10分 / 1ゲーム**
- 目安ターン数：6〜10ターン
- 1ターンの所要時間：約30〜60秒（Acquire + Assign + Resolve）
- バランス調整の基準値：HP=15、DMG={2,3,5,8,12}

---

## プレイヤー

- 人数：2人（Player1 / Player2）
- 対戦形式：AI対戦 / オンライン対戦
- 初期HP：15
- 所持リソース上限：6個

### AI難易度
Easy / Normal / Hard（タイトル画面で選択）

> ローカル対戦は廃止。Assignフェーズの相手スロット非公開ルールが1端末共有では機能しないため。

### 勝利条件
相手のHPを0にしたプレイヤーの勝利。

## リソース

| 種類 | 色 | 略称 | 段が埋まった時の効果 |
|------|-----|------|---------------------|
| 攻撃（Attack） | 赤 | ATK | 相手にダメージを与える |
| 防御（Defense） | 青 | DEF | 自分にシールドを付与する |
| 妨害（Disrupt） | 紫 | DIS | 相手の段をランダムにロック（次ターンのAssign・Resolve両方に影響） |

### 段別の効果量

| 段 | 枠数 | 攻撃ダメージ | 防御シールド量 | 妨害ロック段数 |
|----|------|------------|--------------|--------------|
| 1段目 | 1 | 2 | 1 | 1段 |
| 2段目 | 2 | 3 | 2 | 2段 |
| 3段目 | 3 | 5 | 3 | 3段 |
| 4段目 | 4 | 8 | 4 | 4段 |
| 5段目 | 5 | 12 | 6 | 5段 |

### 妨害の詳細
空き枠がある段の中からランダムに選択してロック。プレイヤー操作なし。
ロックは**次のターンのAssignフェーズ〜Resolve開始直前まで**持続する。

- ロック中の段には**アサインできない**（GetAvailableCount=0）
- ロック中の段は**完成できないため発動しない**（CanAssign=falseで埋められない）
- ロックは次のターンのResolve開始時に自動クリアされる

### シールドの詳細
- 発動するとシールドカウンターが加算される（上限なし）
- 攻撃ダメージを受けた場合、シールドが先に削られる（HP被ダメ前にシールドを消費）
- **Resolveフェーズ終了後にシールドが半減する（端数切り捨て）**
  - 積んだターンのResolveでは全量が有効。次のターン以降は半減した値が持続
  - 例：シールド3 → 1、シールド4 → 2
- シールド値はHP表示の右隣にテキストで常時表示する（"シールド 0" 形式。HPバーは非表示）

---

## リソースプール（中央）

- 3つのプール（Pool0 / Pool1 / Pool2）
- 各プールは最大3枚。プレイヤーが選択すると2枚取得（残り1枚なら1枚のみ）
- Resolveフェーズ終了後に全プールをTopUp補充（不足分のみ最大3枚まで）

---

## フェーズ構成（1ターン）

```
フェーズ1：獲得フェーズ（AcquireP1 → AcquireP2）
フェーズ2：配置フェーズ（AssignP1 → AssignP2）
フェーズ3：発動フェーズ（Resolve）
次のターンへ（ターン数+1）
```

### フェーズ1：リソース獲得
※シールド半減はResolveフェーズ終了後に実施（フェーズ3参照）
- **先手順序ルール：ゲーム開始時にランダムで先手を決定。奇数ターンは初期先手プレイヤーが先、偶数ターンは相手が先**
- 先手 → 後手 の順にプールを選択
- 3つのプールから1つ選択、2枚取得（残り1枚なら1枚）

### フェーズ2：リソース割り当て
- **同時コミット方式**（先後なし）：両プレイヤーが独立して配置を決定し、両者がEnd Assignを宣言した時点で一斉確定
- 内部実装はAssignP1 → AssignP2の順で処理するが、AIは先手でも計画を保留し人間のEndAssign時に一瞬で適用することで同時コミットを実現
- 相手のスロットは常に表示（前ターンまでの配置状態は見える）
- ただし同時コミット方式のため、今ターンの新しい配置はEnd Assign押下まで相手には見えない
- Resolve開始直前に両者の今ターン配置が一斉公開される
- リソース種別を選択 → 段ボタンで配置
- 同じ段には同じ種類のみ配置可能
- ロックされた枠には配置不可
- Resetボタン：このターンの配置をやり直せる
- End Assignボタン：配置終了を宣言
- 未配置リソースは次ターン持ち越し（上限6個）

### フェーズ3：自動解決
- 1段目から順に処理し、各段でP1・P2が同時発動
- 同時発動時の処理順：**Defense（シールド付与）→ Attack / Disrupt** の順に処理
  - 例：P1 Attack・P2 Defense が同じ段で発動した場合、P2のシールドが先に付与され、P1のAttackダメージをシールドが吸収できる
- 両者がともにAttackの場合はDRAW（両者同時にダメージ）
- **ATK vs DEF が同じ段で同時発動した場合はジャストガード（Just Guard）が発動**

### ATKコンボ
- 同ターンにATKが**2段以上**完成して発動した場合にコンボ
- 追加ダメージ = コンボ数（完成したATK段の数）
- 通常ダメージと同じ扱い（シールドで吸収される）
- コンボボーナスは全Row処理後に一括適用

| 状況 | コンボ追加ダメージ |
|---|---|
| ATK 1段のみ | なし |
| ATK 2段完成 | +2 |
| ATK 3段完成 | +3 |
| ATK 4段完成 | +4 |
| ATK 5段完成 | +5 |

### DISコンボ
> **廃止済み（2026-04-07）**  
> DISは何段完成しても常に1ターンのみロック。lockTable {1,2,3,4,5} は各段のロック段数を示す。

### 同時発動の処理順序
同じ段で両者が同時に完成した場合、以下の優先順で処理する。

**基本ルール：DEF は必ず ATK・DIS より先に処理される**

| P1 | P2 | 処理順 | 備考 |
|---|---|---|---|
| ATK | ATK | P1 ATK → P2 ATK | DRAW（両者ダメージ） |
| ATK | DEF | ① P2 DEF先 → ② P1 ATK反射(2倍) | Just Guard |
| ATK | DIS | P1 ATK → P2 DIS | P2ダメージ → P1ロック |
| DEF | ATK | ① P1 DEF先 → ② P2 ATK反射(2倍) | Just Guard |
| DEF | DEF | P1 DEF → P2 DEF | 両者シールド |
| DEF | DIS | P1 DEF先 → P2 DIS | P1シールド後にP1がロック |
| DIS | ATK | P1 DIS → P2 ATK | P2ロック後にP1がダメージ |
| DIS | DEF | P2 DEF先 → P1 DIS | P2シールド後にP2がロック |
| DIS | DIS | P1 DIS → P2 DIS | 両者ロック |

> Just Guard は ATK vs DEF の場合のみ発動。DEFが先にシールドを張ることで成立する。
> ATKより先にDEFを処理しないとJust Guardが発動しないため、この順序は変更禁止。

### Just Guard

- 同じ段でATKとDEFが同時発動したときに発生
- DEFのシールド付与は通常通り発動
- ATKのダメージは無効化され、攻撃側に**2倍のダメージ**が反射される
  - 例：3段目（ATK=4）でJust Guard → 攻撃側に8ダメージ
- 発動後、その段はクリア・全ロック解除
- 解決後に全プールTopUp補充
- **Resolve終了後に両プレイヤーのシールドを半減（切り捨て）**
  - 積んだターンのResolveでは全量が有効。次ターン開始時には半減済みの値になっている

## モード別の仕様差異

### Assignフェーズの情報公開

| モード | Assign中の相手スロット | 備考 |
|---|---|---|
| Local | **隠れる**（全モード共通） | 画面を共有しているが隠蔽で読み合いを担保 |
| AI対戦 | **隠れる** | AIは相手の配置を参照せずに決定（同時コミット相当） |
| Online（予定） | **隠れる** | サーバー経由で確定後に公開（同時コミット） |

### Acquireフェーズの順序

全モード共通で**先手 → 後手の順番処理**。オンラインでも同時取得は行わない（競合処理を避けるため）。

## タイトル画面

| ボタン | 動作 |
|--------|------|
| AIと対戦 | AI難易度選択（かんたん/ふつう/むずかしい） |

選択内容はGameSettingsに保存されBattleSceneへ引き継ぎ。



> **【埋まり済み段へのDIS】**  
> 対象段がすでに埋まっている場合、DISは**空振り**（何も起きない）。  
> 今ターンの効果は通常通り発動し、ロックも翌ターンへの持ち越しも発生しない。
