## スクリプト一覧・依存関係

```
GameManager（MonoBehaviour・Singleton）
├── TurnManager              ターン・フェーズ管理
├── ActionResolver           自動解決フェーズの処理
└── PlayerData × 2          各プレイヤーの状態管理
    ├── SlotGrid             スロット全体（半ピラミッド）
    │   └── SlotRow × 5     各段のスロット管理
    └── ResourceHolder       所持リソース管理

ResourcePoolManager          中央プール3つの管理
└── ResourcePool × 3        各プールの状態

GameUINew（MonoBehaviour・Singleton）  ※旧UIManager相当
├── PlayerPanelUI × 2       各プレイヤーのUI
├── PoolPanelUI             中央プールのUI
│   └── PoolRowUI × 3      各プール行のUI
└── ControlPanelUI          操作パネルのUI

GameUI（MonoBehaviour）      旧OnGUI版UI（GameUINewが起動時に無効化）
TitleScreenUI（MonoBehaviour）タイトル画面UI
EffectManager（MonoBehaviour・Singleton）Resolve演出エフェクト管理
```

### TurnManager（pure C# class）
| 変数/イベント | 型 | 説明 |
|---|---|---|
| currentPhase | GamePhase(get) | 現在のフェーズ |
| turnCount | int(get) | 現在のターン数 |
| initialFirstPlayer | int(get) | ゲーム開始時にランダム決定した先手プレイヤーインデックス |
| FirstPlayerIndex | int(get) | 現在ターンの先手インデックス（奇数=initialFirst、偶数=相手） |
| SecondPlayerIndex | int(get) | FirstPlayerIndexの逆 |
| OnPhaseChanged | Action<GamePhase> | フェーズ遷移時 |

| メソッド | 説明 |
|---|---|
| Initialize(int firstPlayer) | 先手プレイヤーをランダム決定して設定 |
| NextPhase() | 次フェーズへ（Resolve→AcquireP1でturnCount++） |
| SetGameOver() | GameOverフェーズへ強制遷移 |

### ActionResolver（pure C# class）
効果量テーブル（rowIndex 0〜4）
- ダメージ: 1, 2, 4, 6, 10
- シールド: 1, 2, 3, 4, 6
- ロック段数: 1, 2, 3, 4, 5  ※発動段と同数の段をロック（Option B）

| メソッド | 説明 |
|---|---|
| PrepareResolve(p1, p2) | Resolve前にATKコンボ数・DIS持続ターン数を事前計算してフィールドに保存 |
| P1AtkCombo / P2AtkCombo | int(get) 今ターンの各プレイヤーのATKコンボ数 |
| Resolve(PlayerData p1, p2) | 同期版（テスト用）。PrepareResolve→TickLocks→Row1〜5処理→ClearAfterResolve |
| ResolveRowAt(int rowIdx, p1, p2) | 1段分を処理して演出テキストを返す。ATK vs DEF 同時発動時はJust Guard（2倍反射） |
| ClearAfterResolve(p1, p2) | Resolve後にClearCompletedRows（ロックはクリアしない） |
| TickLocksBeforeResolve(p1, p2) | Resolve開始時にロックをカウントダウン。0になった段だけ解除 |
| ApplyLocks(PlayerData, int count, int duration) | ランダムな段にロック付与（durationターン持続） |


### AIController（MonoBehaviour・Singleton）
> `Assets/Scripts/Core/AIController.cs`
> BattleScene の AIController GameObject にアタッチ。
> `GameSettings.Mode != AI` のとき自動的に `enabled = false` になる。

| 定数 | 値 | 説明 |
|---|---|---|
| aiPlayerIndex | 1 | AIは常にPlayer2（index 1）を担当 |

| メソッド | 説明 |
|---|---|
| Start() | GameManager.OnPhaseChangedを購読。初回フェーズを手動チェック |
| CaptureOppSnapshot(opp) | AssignP1開始時に相手スロットをスナップショット保存（公平性保証） |
| OnPhaseChanged(GamePhase) | AIのターン（CurrentActorIndex==1）のときにDoAcquire/DoAssignを起動 |
| DoAcquire() | 0.8秒待機後にプール選択。Easy=ランダム / Normal・Hard=枚数最大を優先 |
| DoAssign() | 0.8秒待機後に配置。EndAssign()を呼んでターン終了 |
| AssignEasy(gm, ai) | 所持リソースをシャッフルしてランダムな段へ配置 |
| AssignNormal(gm, ai) | ATK→DEF→DIS の優先順で完成に近い段へ配置 |
| AssignHard(gm, ai, opp) | 相手がATKを積んでいればDEF優先→残りはNormalと同じ |

### AnnouncementUI（MonoBehaviour・Singleton）
> `Assets/Scripts/UI/AnnouncementUI.cs`
> BattleScene の AnnouncementOverlay にアタッチ。ゲーム開始・フェーズ開始の演出を管理する。
> Awake() 時点で `IsAnimating = true` にセットし、Start() で ShowGameStart() を自動実行する。

| フィールド | 型 | 説明 |
|---|---|---|
| overlayGroup | CanvasGroup | アニメーション中は alpha・blocksRaycasts を制御 |
| mainTextRect | RectTransform | GAME START のスケールアニメーション用 |
| mainText | TextMeshProUGUI | メインアナウンステキスト |
| subText | TextMeshProUGUI | サブテキスト（Turn N 等） |
| IsAnimating | bool(get) | アニメーション中は true。AI・ResolveCoroutine がこれを待つ |

| メソッド | 説明 |
|---|---|
| ShowGameStart() | GAME START 演出（スケール+フェード、約1.75s） |
| OnPhaseChanged(GamePhase) | AcquireP1/AssignP1/Resolve のみアナウンス。Resolve は1回のみ |

| フェーズ | テキスト | 色 | 表示時間 |

| ゲーム開始 | GAME START | ゴールド | 約1.75s |
| AcquireP1 | ACQUIRE + Turn N | 白 | 約1.36s |
| AssignP1 | ASSIGN | 白 | 約1.36s |
